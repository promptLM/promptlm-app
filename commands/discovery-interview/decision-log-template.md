# Decision Log — Running Template

Maintain this log throughout the interview. After each answer, append or
update the relevant fields. Show the user the updated log on request, and at
each phase boundary.

```
## Decision being considered
<one-sentence statement of what the user is deciding>

## User objective
<what success looks like, in the user's own words>

## Stakeholders
- <who is affected, who decides, who must be consulted>

## Constraints
- <budget, deadline, regulatory, contractual, ethical, capacity>

## Facts gathered
- <fact> — source: <user statement / document / system>
- ...

## Assumptions
- <assumption — clearly labeled as such, not a fact>
- ...

## Options
1. <option name>
   - Upsides: ...
   - Downsides: ...
   - Dependencies: ...
   - Effort/cost (qualitative): ...
2. <option name>
   - ...

## Risks
- <risk>:
  - What: ...
  - Why it matters: ...
  - Consequence if ignored: ...
  - Mitigation / diligence: ...
  - Expert review needed: yes/no — <kind of expert>

## Unknowns / Open questions
- <question> — needed for: <which decision criterion or option>
- ...

## Decision criteria
- <criterion>: <how we'll judge options against it>
- ...

## Tradeoffs
- <axis A> vs <axis B>: <user's stated preference, if any>
- ...

## Evidence still needed
- <evidence>: <why it matters, who would provide it>
- ...

## Preliminary reasoning
<short narrative connecting facts → criteria → leading option(s); explicitly
labeled as preliminary, subject to remaining unknowns>

## Final recommendation
<filled in only at Phase 5>

## Follow-up actions
- <action>: <owner> — <by when, if discussed>

## Expert review items
- <item>: <kind of expert needed and what to ask them>
```

Rules:

- Never delete an entry; supersede it with a dated update if the user revises.
- Every assumption must remain visible — never silently promote one to a fact.
- Mark every "fact" with its source so the user can verify later.
