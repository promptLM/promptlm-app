How to read a surface page

Each surface page has four sections in this order:

  1. Screen reference — link to the live mock in handoff/webui/ or handoff/report/, plus a one-paragraph design intent.
  2. API operations — what the screen calls on load, save, action.
  3. Schema bindings — every visible UI field, its source schema property, type, and required-ness.
  4. Notes — validation rules, error states, what's mocked vs real.
Caveats carried from HANDOVER.md. Production observability (p95, success rate, cost) is fake on the report and detail pages — feature-flag in production. The executions[] table assumes a future promptlm run capture pipeline. Evaluations are design-only — runner doesn't exist yet.
