Local state & reset semantics (Q5 lock)

  • executions[] resets on any change to request. Triggers: model, snapshot, parameters, messages content, placeholder list shape (renames / adds / removes — not value edits).
  • Placeholder value edits do not reset history — they're inputs, not part of the request shape. Each run snapshot still carries the values used.
  • Request-changed banner appears in the history strip when the user has edited the request shape since the last run; clicking "Clear" purges executions[] explicitly.
  • Persistence: placeholder edits live in component state until the user clicks "Save placeholders" (Q3 — explicit save, no auto-save). Run results are written back to executions[] by the server on every Run.
