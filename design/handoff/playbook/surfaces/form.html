Notes & known issues

  • Stripped from form UI: version, revision, repositoryUrl, request.url. Kept in payload but never rendered as inputs — surface in the sticky header context strip.
  • Mode differences are minimal: empty draft vs loaded; CTA label "Create" vs "Save"; Identity collapsed by default in edit, expanded in create.
  • Error count badges on each collapsible — auto-expand the section if errors exist on save attempt.
  • Eval section renders disabled with a "Available with promptLM Pro" lock until Evaluation type has a real shape.
  • Tools section persists into extensions['x-promptlm.tools'] — not a first-class field on PromptSpecCreationRequest yet.
  • Open bugs: vendor select shows "— select —" even with value set; create-mode name placeholder reads like a real value. See HANDOVER §5.
  • Editor / Test tab strip (#95) sits below the sticky header. The Editor tab is everything documented above; the Test tab has its own playbook page — see Test tab. Release flow (button + side rail + state machine + gates) is documented in Release flow.
