promptLM User Documentation

This page is the initial end-user documentation draft for promptLM. It focuses on installation, first-run workflows, and the core commands required to create and release prompts.

promptLM is in early alpha. Commands and workflow details may change in upcoming releases.

Get Started

Prerequisites

  • Git installed locally.
  • A GitHub account and repository access for remote storage workflows.
  • macOS, Linux, or Windows shell access.

Install promptlm-cli (macOS / Linux)

curl -fsSL https://raw.githubusercontent.com/promptLM/promptlm-app/main/scripts/install.sh | bash

Install promptlm-cli (Windows PowerShell)

$script = irm https://raw.githubusercontent.com/promptLM/promptlm-app/main/scripts/install.ps1
& ([scriptblock]::Create($script))

Verify installation

promptlm-cli --version

Core CLI Workflow

Use these commands to connect a prompt repository, create a prompt draft, inspect it, and request a release.

1. Create a new prompt repository

promptlm-cli repo create --dir ./workspace --name your-org/your-prompt-repo

Alternative: clone an existing repository instead of creating one.

promptlm-cli repo clone --repo https://github.com/your-org/your-prompt-repo.git --target ./workspace

2. Create a prompt draft

promptlm-cli prompt create --name sentiment-check --group support --userMessage "Analyze the sentiment of: {text}" --placeholder text="I love promptLM"

The command returns the prompt id. Keep it for the next commands.

3. Show the latest prompt version

promptlm-cli prompt show --id <prompt-id>

4. Request a release

promptlm-cli prompt release --id <prompt-id>

If your project uses two-phase promotion, complete the release after the pull request is merged.

promptlm-cli prompt release complete --id <prompt-id> --pr <pr-number>

PromptLM Studio

Start the local Studio web app from the CLI:

promptlm-cli studio --port 8085

To start without opening a browser automatically:

promptlm-cli studio --port 8085 --no-browser true

Default URL: http://127.0.0.1:8085/

Troubleshooting

  • If promptlm-cli is not found, add the install directory to your shell PATH.
  • If Studio startup reports a used port, rerun with a different port (for example --port 8090).
  • If release commands fail, verify repository connectivity and GitHub permissions.
  • For product issues and bugs, use the GitHub Issues tracker linked below.
